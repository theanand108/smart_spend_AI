# Claude UI Rules

Read before implementing any UI changes.

---

Never redesign the entire application.

Improve only the requested section.

---

Never invent new features.

Never remove functionality.

Never change product messaging.

---

Read these documents first:

1. PRODUCT_DESIGN_DOCUMENT.md

2. CLAUDE_CONTEXT.md

3. DESIGN_PHILOSOPHY.md

4. UI_PRINCIPLES.md

---

Follow these inspirations:

Primary

CRED

Secondary

Mercury

Interactions

Arc Browser

Interpret.

Never imitate.

---

Avoid:

- giant hero sections

- oversized typography

- excessive gradients

- glassmorphism

- emojis in headings

- excessive pills

- decorative animations

- meaningless statistics

- fake testimonials

---

Prefer:

- premium spacing

- restrained typography

- subtle shadows

- smooth transitions

- information hierarchy

- clean dashboards

---

Do not replace layouts unless requested.

Refine.

Don't redesign.

---

Always explain:

Why is this better?

What usability problem does it solve?

How does it align with the Design Philosophy?

Wait for approval before implementation.

---

One section at a time.

Never polish the whole website in one prompt.

## Existing Implementation Check

Before implementing a requested sprint:

1. Verify whether the requested work is already present in the codebase.
2. If it is already implemented, do not modify the files.
3. Explain your findings and ask whether to proceed to the next sprint instead.
4. Avoid rewriting completed work unless explicitly requested.
